# Security Policy

## 安全漏洞披露

Ottomi Nexus 多模态 AI 数据中台高度重视产品安全与数据合规，若您发现产品存在安全漏洞、数据风险、授权隐患或其他合规相关安全问题，**请勿通过 GitHub Issues、公共社区等公开渠道披露**，避免漏洞被恶意利用。

请通过官方商务邮箱提交完整漏洞信息，包含：漏洞类型、复现步骤、影响范围、风险等级及相关佐证材料，我们将第一时间进行核查与处理。

## 官方安全联系渠道

商务咨询邮箱：info@oceandatum.com

## 漏洞处理流程

1. 收到安全漏洞反馈后，我们将在**5 个工作日内**完成初步核查，并向您反馈受理结果；
2. 针对确认的安全漏洞，官方技术团队将制定修复方案，同步评估风险影响，开展紧急修复；
3. 漏洞修复完成后，将通过产品版本更新、官方通知等方式同步，核心安全修复将优先推送至商业版用户；
4. 全程对漏洞反馈信息严格保密，未经允许不向第三方泄露。

## 安全使用提示

1. 请用户严格按照官方部署指南安装部署，切勿自行修改产品授权机制、安全配置及核心底层代码；
2. 定期更新至官方最新版本，及时获取安全修复与性能优化内容；
3. 遵守《数据安全法》《个人信息保护法》等相关法律法规，合规使用产品处理数据；
4. 严禁未经授权破解、反编译、篡改产品安全模块，相关违规行为需自行承担法律责任。

## 第三方组件安全

本产品集成的第三方开源组件，均遵循对应开源协议，官方会定期核查第三方组件安全漏洞，及时升级迭代规避风险，详细第三方组件清单及协议说明请查看项目`NOTICE`文件。



# Security Policy

## Security Vulnerability Disclosure

Ottomi Nexus Multimodal AI Data Platform takes product security and data compliance seriously. If you discover a security vulnerability, data risk, authorization flaw, or any other compliance-related security issue, **please do not disclose it through public channels such as GitHub Issues or community forums**, to prevent the vulnerability from being exploited maliciously.

Please submit complete vulnerability details to our official business email, including: vulnerability type, reproduction steps, scope of impact, severity level, and any supporting materials. We will investigate and respond as soon as possible.

## Official Security Contact

Business email: info@oceandatum.com

## Vulnerability Handling Process

1. Upon receiving a security vulnerability report, we will complete an initial review within **5 business days** and notify you of the acceptance result;
2. For confirmed vulnerabilities, the official technical team will develop a remediation plan, assess the risk impact, and carry out emergency fixes;
3. Once a fix is complete, it will be communicated via product version updates and official announcements — critical security fixes will be prioritized for Commercial Edition users;
4. All vulnerability reports will be kept strictly confidential and will not be disclosed to any third party without permission.

## Security Usage Guidelines

1. Follow the official deployment guide strictly for installation and deployment. Do not modify the product's licensing mechanism, security configurations, or core underlying code;
2. Keep the product updated to the latest official version to receive timely security fixes and performance improvements;
3. Comply with applicable laws and regulations, including the Data Security Law and the Personal Information Protection Law, when using the product to process data;
4. Unauthorized cracking, decompilation, or tampering with the product's security modules is strictly prohibited. Any such violations are subject to legal liability.

## Third-Party Component Security

All third-party open-source components integrated in this product comply with their respective open-source licenses. The official team regularly audits third-party components for security vulnerabilities and upgrades them in a timely manner to mitigate risks. For a detailed list of third-party components and their license information, please refer to the `NOTICE` file in the project repository.
