# 数据质检规则库详情
本文档汇总了Ottomi Nexus平台内置的全量数据质检规则，基于DAMA数据质量标准设计，按业务场景分为单表结构检查、单表字段内容检查、单表条件检查、多表/全库结构检查、多表动态检查、实时数据检查六大类，可全面覆盖通用数据合规校验、业务逻辑校验、实时数据质量监控等全场景需求。允许用户自定义质检规则，并且随着软件的升级迭代，将会支持更多的规则。

## 1. 单表结构检查规则
| 规则名称 | 规则说明 | 质量属性 |
|---------|---------|---------|
| 表非空检查 | 判断目标表是否为空表，无数据记录 | 完备性 |
| 时间戳字段检查 | 判断表中是否包含合规的时间戳字段，用于数据更新时间追溯 | 准确性 |
| 字段注释齐全检查 | 判断表中所有字段是否均补充了业务注释，无缺失 | 完备性 |
| 主键完整性检查 | 判断表中是否设置了主键约束，确保数据行唯一标识 | 完整性 |
| 重复数据检查 | 判断表中是否存在完全重复的记录行 | 准确性 |
| 引用完整性检查 | 判断表中是否存在主外键关联关系，确保关联字段引用完整无缺失 | 合理性 |
| 最后更新时间合规检查 | 校验表中数据的最后更新时间是否符合业务预设的时间条件 | 及时性 |
| 数据增量存在性检查 | 校验目标周期内数据是否有新增增量，无增量则触发异常 | 准确性 |
| 数据增量异常检查 | 校验数据增量幅度是否超出业务预设的正常区间，异常则触发告警 | 准确性 |

## 3. 单表条件检查规则
| 规则名称 | 规则说明 | 质量属性 |
|---------|---------|---------|
| 多要素耦合检查 | 用于校验同一条记录中多个字段之间的业务逻辑关系，支持灵活的“与/或”逻辑组合。可配置“检查字段”（待验证字段）与“规则字段”（定义验证规则的字段），支持范围类（大于、小于、大于等于、小于等于、Null/Not Null等）、枚举类（自定义枚举值、同表字段group by结果）两种校验规则 | 准确性 |
| 监测时间与当前时间比较检查 | 校验数据中的监测时间字段与当前系统时间的大小关系，支持大于、小于等预设条件判断 | 准确性 |
| 字段值非负检查 | 校验数值型字段值是否大于等于0，排除负数异常值 | 合理性 |
| 多字段复合逻辑检查 | 支持同表内多个字段的复杂业务逻辑组合校验，满足多条件联动的业务校验需求 | 准确性 |
| 变量值域检查 | 支持将同表内其他字段值作为校验区间的动态变量，实现字段值的动态值域校验 | 合理性 |

## 5. 多表动态检查规则

| 数据内容一致性检查 | 校验跨表关联的字段内容是否完全一致，无数据差异 | 一致性 |

## 6. 实时数据检查规则
| 规则名称 | 规则说明 | 质量属性 |
|---------|---------|---------|
| 实时数据推送时效性检查 | 持续监测目标实时数据流的更新频率，当关键指标的“最近一次数据上报时间”与当前时间的差值超过预设的静默容忍时长时，自动判定数据流中断并生成异常告警事件。可自定义监测指标、时间阈值、告警等级 | 及时性 |
| 窗口聚合与状态关联复合检查 | 在设定的时间窗口内，对目标指标进行统计聚合（如累计值、最大值、最小值等），同时支持结合该对象的实时状态进行复合逻辑判断（支持与/或关系），满足组合条件时自动触发对应等级的业务事件与告警 | 及时性 |
